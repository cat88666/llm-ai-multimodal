# 批量转换技能

## 触发条件

用户消息中出现以下任意意图时，立即触发本技能，无需用户进一步确认：

- "帮我把 `<路径>` 所有文件转化为 md"
- "把 `<路径>` 所有文件全部转成 md"
- "转换 `<路径>` 下的文件"
- "继续转换"、"上次中断了，继续"
- `/批量转换 <路径>`

**路径识别规则：**
- 相对路径（如 `raw/test`）：以本工具根目录为基准
- 绝对路径（如 `/Users/mac/ai/llm-ai/llm-ai-qingan/raw/一审`）：直接使用

---

## 执行流程（逐步完成，不等待用户确认）

### 第一步：提取目录路径

从用户消息中识别目录路径。
- 找到第一个像路径的字符串（含 `/` 或以文件夹名结尾）
- 判断是相对路径还是绝对路径
- 如果消息中没有路径，询问用户："请提供待转换的目录路径"

### 第二步：检查环境（静默执行，失败才报告）

```bash
python3 -c "import anthropic, PIL, fitz, docx; print('依赖正常')"
```

如果失败，执行：
```bash
pip install -r requirements.txt
```

### 第三步：执行转换

```bash
python3 skills/批量转换/scripts/convert.py <用户提供的路径>
```

**关键行为：**
- 脚本会自动推导输出目录（无需用户指定）
- 自动读取 `output/output.log`，跳过已成功完成的文件
- 实时在终端显示进度，同时写入日志

### 第四步：如果上次中断，继续执行

直接再次运行相同命令即可：
```bash
python3 skills/批量转换/scripts/convert.py <同一路径>
```

脚本会自动识别日志中的已完成记录，从中断处继续。
**不需要加 `--force`，不需要手动清理。**

如果需要完全重新转换（忽略历史记录）：
```bash
python3 skills/批量转换/scripts/convert.py <路径> --force
```

### 第五步：汇报结果

转换结束后，读取 `output/output.log` 最后一段，向用户报告：
- ✓ 成功转换了哪些文件
- ✗ 失败的文件（如有）及原因
- 输出目录位置

## 输出目录推导规则

| 输入路径 | 自动推导的输出目录 |
|---------|----------------|
| `raw/test` | `<本工具目录>/output/test` |
| `raw/test/子目录` | `<本工具目录>/output/test/子目录` |
| `/path/to/其他项目/raw/一审` | `/path/to/其他项目/output/一审` |
| `/path/to/其他项目/raw/二审` | `/path/to/其他项目/output/二审` |

**断点续传原理：**
每条成功日志末尾记录了源文件的绝对路径（`| /绝对路径/...`）。
下次运行时脚本解析这些路径，自动跳过，从未完成处继续。

## 技能协作关系

本技能作为调度层，不直接处理文件内容，调用链如下：

```
批量转换（调度）
  ├── .pdf  → skills/pdf转md/scripts/pdf_to_md.py
  ├── .docx → skills/word转md/scripts/word_to_md.py
  ├── .jpg  → skills/图片转md/scripts/img_to_md.py
  ├── .jpeg → skills/图片转md/scripts/img_to_md.py
  ├── .png  → skills/图片转md/scripts/img_to_md.py
  └── .txt  → skills/文本转md/scripts/txt_to_md.py
```

每个格式技能独立维护，互不影响。

## 常见问题

**依赖缺失** → `pip install -r requirements.txt`

**目录路径不存在** → 检查路径拼写，确认目录存在

**上次中断，如何继续** → 直接运行同一命令，无需任何额外操作

**想重新转换所有文件** → 加 `--force` 参数
