# 文件转 Markdown 任务规划

## 触发识别

用户消息中出现 **路径 + 转换/转化 + md/Markdown** 的任意组合时触发，例如：

- "把 `raw/test` 所有文件转化 md"
- "把 `/Users/mac/.../raw/test` 所有文件转成 md"
- "把 `raw/test` 所有文件转化为 md，生成后放到 `/output/结果`"

## 参数提取

**源目录**（必须）：从用户消息中识别路径字符串。
- 以 `/` 开头 → 绝对路径，直接使用
- 否则 → 相对路径，以本工具根目录为基准

**目标目录**（可选）：用户说"放到 XXX"时提取；未指定则交由脚本自动推导。

## 执行

参数清晰时**直接执行，无需等待用户确认**：

```bash
python3 skills/批量转换/scripts/convert.py <源目录>
# 指定目标时：
python3 skills/批量转换/scripts/convert.py <源目录> --output <目标目录>
```

## 异常处理

| 情况 | 处理 |
|------|------|
| 路径不存在 | 告知用户，不执行 |
| 依赖缺失 | 执行 `pip install -r requirements.txt` 后继续 |
| 上次中断 | 直接运行同一命令，脚本自动断点续传 |
| 想重新全量转换 | 加 `--force` |

## 完成后汇报

读取 `output/output.log` 最后一段，向用户报告成功数、失败文件（如有）、输出目录位置。
